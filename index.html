Architecture

Atomes

Étude des principales plateformes. Déduction de 3 atomes.

  • Utilisateurs
  • Conversations
  • Messages

Utilisateurs

  • ID
  • Nom (au moins un prénom ou un pseudo)
  • Langue (souvent)
  • Time zone (des fois)
    • Ça compte car le rendu est côté serveur

Conversation

  • ID
  • 1 à plusieurs participants (liste inconnue)

Message

  • Émetteur
  • Destinataire
  • Contenu (approche en calques)

Infos pratiques

  • Licence AGPL v3+
    • Les améliorations remontent upstream
    • Vos API ne sont pas contaminées
  • Python 3.6+ (basé sur asyncio)
  • Nécessite un Redis

Faire le flow

Créer les états

class S001xWelcome(NumberBotState):
    @page_view('/bot/welcome')
    async def handle(self) -> None:
    name = await self.request.user.get_friendly_name()

    self.send(
        lyr.Text(t('WELCOME', name=name)),
        fbl.QuickRepliesList([
            fbl.QuickRepliesList.TextOption(
                slug='play',
                text=t.LETS_PLAY,
                intent=its.LETS_PLAY,
            ),
        ]),
    )

Configurer les transitions


transitions = [
    Tr(
        dest=S001xWelcome,
        factory=trg.Action.builder('get_started'),
    ),
    Tr(
        dest=S002xGuessANumber,
        origin=S001xWelcome,
        factory=trg.Choice.builder('play'),
    ),
]
